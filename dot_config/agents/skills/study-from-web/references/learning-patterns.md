# 단계별 학습 콘텐츠 생성 패턴

## 개요

탐색된 문서를 기초/중급/고급 단계로 나누어 학습 콘텐츠를 생성한다.
각 단계는 고유한 구성 패턴을 가지며, 사용자의 이해를 점진적으로 확장한다.

---

## 🟢 기초 (Beginner) 패턴

### 목표
"이것이 무엇이고, 왜 필요하고, 어떻게 시작하는가"를 이해하게 한다.

### 구성 순서

#### 1. 정의 (What)
- **한 문장**으로 핵심 정의를 제시한다
- 기술 용어는 원어 병기: "리액트 훅 (React Hook)은..."
- 예시: "리액트 훅 (React Hook)은 함수형 컴포넌트에서 상태와 생명주기 기능을 사용할 수 있게 해주는 함수입니다."

#### 2. 비유 (Why it matters)
- 일상적인 비유로 핵심 개념을 설명한다
- 비유 → 실제 기술 개념으로 자연스럽게 연결
- 예시: "훅은 레고 블록과 비슷합니다. 필요한 기능을 하나씩 골라서 컴포넌트에 끼워 넣는 것이죠."

#### 3. 핵심 용어 (Key Terms)
- 표(table) 형식으로 3~5개 용어를 정리한다

```markdown
| 용어 | 원어 | 설명 |
|------|------|------|
| 상태 | State | 컴포넌트가 기억하는 데이터 |
| 이펙트 | Effect | 렌더링 후 실행되는 부수 효과 |
```

#### 4. 기본 사용법 (How to start)
- **가장 간단한** 코드 예시를 보여준다
- 코드에 한국어 주석을 반드시 포함한다
- 한 줄 최대 150자를 지킨다

```javascript
// 상태를 사용하는 가장 간단한 예시
const [count, setCount] = useState(0); // 초기값 0으로 상태 생성
```

#### 5. 주의사항 (Common Pitfalls)
- 초보자가 자주 하는 실수 2~3가지를 나열한다
- "❌ 이렇게 하지 마세요" / "✅ 이렇게 하세요" 형식

## 🟡 중급 (Intermediate) 패턴

### 목표
"더 깊이 이해하고, 실제 프로젝트에서 활용할 수 있다"

### 구성 순서

#### 1. 기초 복습 (Quick Recap)
- 선수 지식을 **한 줄**로 요약한다
- 예시: "💡 복습: useState는 컴포넌트에 상태를 추가하는 훅입니다."

#### 2. 심화 개념 (Deep Dive)
- 동작 원리를 더 상세히 설명한다
- "내부에서 어떻게 작동하는가"에 초점
- 다이어그램이나 단계별 설명 활용

```
실행 흐름:
1. 컴포넌트 렌더링 → 2. 훅 호출 → 3. 상태 업데이트 → 4. 리렌더링
```

#### 3. 실용 패턴 (Practical Patterns)
- 실제 프로젝트에서 자주 쓰는 패턴 2~3가지
- 각 패턴에 이름을 붙이고, 사용 시나리오를 설명한다
- 코드 예시에 한국어 주석 포함

#### 4. 개념 비교 (Comparison)
- 유사한 개념들을 비교하는 표를 만든다

```markdown
| 기준 | useState | useReducer |
|------|----------|------------|
| 적합한 경우 | 단순한 상태 | 복잡한 상태 로직 |
| 상태 업데이트 | 직접 설정 | 액션 기반 |
```

#### 5. 실습 과제 (Practice)
- 직접 해볼 수 있는 작은 과제를 제시한다
- 요구사항을 명확히 제시하고, 힌트를 제공한다
- 예시: "할 일 목록에 항목 추가/삭제 기능을 만들어보세요."

## 🔴 고급 (Advanced) 패턴

### 목표
"전문가 수준의 이해와 실전 적용 능력을 갖추다"

### 구성 순서

#### 1. 배경 지식 (Context)
- 이 고급 개념이 **왜 필요한지** 배경을 설명한다
- 기존 방식의 한계점을 먼저 제시
- 예시: "컴포넌트가 많아지면 상태 관리가 복잡해집니다. 이때..."

#### 2. 문제 정의 (Problem Statement)
- 해결하려는 구체적인 문제를 정의한다
- 코드 예시로 "문제 상황"을 보여준다

#### 3. 해결 방법 (Solution)
- 단계별로 상세한 구현 방법을 설명한다
- 코드 예시에 한국어 주석을 포함한다
- 각 단계의 "왜?"를 설명한다

#### 4. 성능 고려사항 (Performance)
- 성능에 미치는 영향을 구체적으로 설명한다
- 벤치마크나 비교 데이터가 있으면 제시
- 최적화 팁을 제공한다

#### 5. 안티패턴 (Anti-patterns)
- 피해야 할 패턴 2~3가지를 "왜 나쁜지"와 함께 설명한다

```
❌ 안티패턴: 매 렌더링마다 새 객체 생성
→ 불필요한 리렌더링을 유발합니다

✅ 올바른 방법: useMemo로 메모이제이션
→ 참조 동일성을 유지하여 불필요한 리렌더링을 방지합니다
```

#### 6. 실전 사례 (Real-world Example)
- 실제 프로젝트에서 어떻게 적용되는지 보여준다
- 가능하면 원문서의 예시를 활용한다
- 코드가 길면 핵심 부분만 발췌하고, 전체 맥락을 설명한다

## 한국어 작성 가이드

### 기술 용어 병기 규칙

- **첫 등장 시**: "상태 관리 (State Management)"로 병기
- **이후**: 한국어만 사용하거나, 맥락에 따라 원어 사용
- **코드 내**: 원어 그대로 사용, 주석은 한국어

### 문장 작성 규칙

- **한 문장에 하나의 개념만** 담는다
- 접속사로 연결하기보다 문장을 나눈다
- ❌ "훅은 함수형 컴포넌트에서 상태를 사용할 수 있게 해주며, 클래스 컴포넌트 없이도 생명주기를 관리할 수 있습니다."
- ✅ "훅은 함수형 컴포넌트에서 상태를 사용할 수 있게 해줍니다. 클래스 컴포넌트 없이도 생명주기를 관리할 수 있습니다."

### 비유 활용 규칙

- 추상적 개념은 **일상 비유로 먼저** 설명한다
- 비유 후 바로 실제 개념으로 연결한다
- 비유가 부적절하면 (지나치게 단순화) 사용하지 않는다

### 격려 메시지

- 어려운 개념 설명 후: "처음에는 복잡해 보일 수 있지만, 하나씩 따라가면 이해할 수 있습니다."
- 실습 과제 전: "부담 없이 시도해보세요. 정답은 하나가 아닙니다."
- 퀴즈 오답 후: "좋은 시도입니다! 다시 한번 생각해볼까요?"

## 섹션 간 연결

각 섹션 시작 시 이전 섹션과의 연결고리를 제공한다:

```
💡 지난 시간에 [이전 주제]를 배웠습니다.
이번에는 그 지식을 바탕으로 [현재 주제]를 알아보겠습니다.
```

이전 섹션의 핵심 개념을 현재 섹션에서 어떻게 확장하는지 한 문장으로 설명한다.

## 콘텐츠 분량 기준

| 단계 | 섹션당 분량 | 코드 예시 | 표 |
|------|-----------|----------|-----|
| 기초 | 5~8분 읽기 | 1~2개 | 1개 |
| 중급 | 8~12분 읽기 | 2~3개 | 1~2개 |
| 고급 | 10~15분 읽기 | 3~4개 | 1~2개 |

분량은 가이드라인이며, 내용에 따라 유연하게 조절한다.
핵심은 "한 번에 소화할 수 있는 양"을 넘지 않는 것이다.
