# 스킬 분석 체크리스트

## 1. 구조 분석

### Frontmatter
- [ ] `name`과 `description` 필드가 존재하는가?
- [ ] `description`이 충분히 구체적인가? (무엇을 하는지 + 언제 트리거되는지)
- [ ] 불필요한 frontmatter 필드가 없는가?

### 파일 구조
- [ ] SKILL.md가 500줄 이하인가?
- [ ] Progressive Disclosure가 적용되었는가? (상세 내용이 references로 분리)
- [ ] 불필요한 파일이 없는가? (README.md, CHANGELOG.md 등)
- [ ] scripts/, references/, assets/ 디렉토리가 적절히 사용되는가?
- [ ] 참조 파일이 1단계 깊이로 유지되는가? (SKILL.md → references/ 직접 참조)

### 리소스 분류
- [ ] 실행 가능한 코드가 scripts/에 있는가?
- [ ] 문서/참조가 references/에 있는가?
- [ ] 출력용 파일(템플릿, 이미지 등)이 assets/에 있는가?
- [ ] 리소스 파일이 SKILL.md에서 적절히 참조되는가?

## 2. 내용 분석

### 명확성
- [ ] 지시사항이 명령형으로 작성되었는가?
- [ ] Claude가 이미 아는 내용을 불필요하게 설명하지 않는가?
- [ ] 구체적인 예제가 포함되었는가?
- [ ] 워크플로우가 명확한 단계로 구분되는가?

### 완성도
- [ ] 주요 사용 시나리오가 모두 커버되는가?
- [ ] 엣지 케이스 처리가 포함되는가?
- [ ] 에러 상황에 대한 안내가 있는가?
- [ ] 결정 포인트(분기)가 명확히 안내되는가?

### 간결성
- [ ] "When to Use This Skill" 섹션이 body가 아닌 description에 있는가?
- [ ] 중복 내용이 없는가? (SKILL.md ↔ references 간)
- [ ] 토큰 효율적인가? (불필요한 설명 제거)

## 3. 의존성 분석

- [ ] Claude 빌트인 도구로 대체 가능한 외부 호출이 있는가?

### 패키지 의존성
- [ ] pip/npm 설치가 필요한 패키지가 있는가?
- [ ] 표준 라이브러리로 대체 가능한 외부 패키지가 있는가?
- [ ] 시스템 바이너리 의존성이 있는가? (ffmpeg, imagemagick 등)

### 환경 의존성
- [ ] 특정 OS에서만 동작하는 부분이 있는가?
- [ ] 특정 환경 변수나 설정 파일이 필요한가?
