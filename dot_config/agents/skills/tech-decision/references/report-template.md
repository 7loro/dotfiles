## 간소화 버전 (Quick Decision)

비교 대상 2개, 비교적 단순한 주제, 또는 사용자가 빠른 결론을 원할 때 사용한다.

```markdown
# Quick Decision: {주제}

## 결론

> **추천: {옵션명}** (신뢰도: {🟢/🟡/🔴})
>
> {한 문장 요약}

## 비교표

| 기준 | {옵션 A} | {옵션 B} |
|------|----------|----------|
| {기준 1} | ⭐⭐⭐⭐ | ⭐⭐⭐ |
| {기준 2} | ⭐⭐⭐ | ⭐⭐⭐⭐⭐ |
| ... | ... | ... |
| **종합** | **X/5** | **X/5** |

## 핵심 근거

1. **{근거 1}**: {설명} ([출처](URL))
2. **{근거 2}**: {설명} ([출처](URL))
3. **{근거 3}**: {설명} ([출처](URL))

## 주의사항

- {리스크 1}: {완화 방안}
- {리스크 2}: {완화 방안}

---

### 출처
- [출처 제목](URL) — 유형
```

## 엣지 케이스 템플릿

### 명확한 승자가 없을 때

```markdown
## 결론

> **조건부 추천** (신뢰도: 🟡 중간)
>
> {조건 A}에서는 **{옵션 A}**, {조건 B}에서는 **{옵션 B}**를 추천합니다.

### 조건별 추천 상세
| 조건 | 추천 옵션 | 이유 |
|------|----------|------|
| {조건 A} | {옵션 A} | ... |
| {조건 B} | {옵션 B} | ... |
```

### 추가 정보가 필요할 때

```markdown
## 결론

> **잠정 추천: {옵션명}** (신뢰도: 🔴 낮음)
>
> 최종 결정 전 아래 사항을 확인하세요.

### 확인 필요 사항
1. [ ] {확인 항목 1} — {왜 필요한지}
2. [ ] {확인 항목 2} — {왜 필요한지}
```

### 모든 옵션이 부적절할 때

```markdown
## 결론

> **현재 옵션 중 명확한 추천이 어렵습니다.**
>
> 대안으로 **{대안 옵션}** 검토를 제안합니다.

### 기각 사유
| 옵션 | 기각 이유 |
|------|----------|
| {옵션 A} | {이유} |
| {옵션 B} | {이유} |

### 대안 제안
- **{대안 옵션}**: {왜 이것이 더 적합할 수 있는지}
```
